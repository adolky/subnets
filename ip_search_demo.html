🔍 IP Address Search Feature

✅ New Feature Added

A powerful IP address search field has been added to the main subnet calculator interface, positioned right below the Network Address/Mask bits section.

🎯 How It Works

🔍 Smart IP Lookup:
Enter any IP address (e.g., 192.168.3.7) and the system will find which subnet contains that IP.
🎨 Visual Highlighting:
The matching subnet row is highlighted with a green border and light yellow background, plus smooth scrolling to the result.
📊 Real-time Validation:
Validates IP address format and provides immediate feedback on search results.
🧮 Subnet Mathematics:
Uses bitwise operations to accurately determine if an IP falls within a subnet range.

💡 Usage Examples

Scenario 1: Network 192.168.1.0/24 (covers 192.168.1.0 to 192.168.1.255)
Search: 192.168.1.100
Result: "Found in subnet: 192.168.1.0/24" + row highlighting
Scenario 2: Network 10.0.0.0/16 (covers 10.0.0.0 to 10.0.255.255)
Search: 10.0.5.23
Result: "Found in subnet: 10.0.0.0/16" + row highlighting
Scenario 3: Search for IP not in any subnet
Search: 172.16.1.1
Result: "IP address not found in any current subnet"
Invalid Input: Malformed IP address
Search: 999.999.999.999
Result: "Invalid IP address format"

🖥️ Interface Location

The search field is positioned strategically between the main network configuration and the subnet table:

Network Address: [192.168.0.0]   Mask bits: /[16]   [Update] [Reset]

🔍 Find IP Address: [e.g. 192.168.3.7_____] [Clear] → Search result appears here

⚡ Technical Implementation

Key Functions Added: • searchIPAddress() - Main search logic with validation • findSubnetForIP() - Scans subnet table for matching range • isIPInSubnet() - Bitwise subnet math for precise matching • highlightSubnetRow() - Visual highlighting with animation • clearIPSearch() - Reset search and highlighting Advanced Features: • Real-time input validation • Smooth scrolling to results • Animated highlighting effects • Support for all subnet sizes (/8 to /30) • Clear error messaging
